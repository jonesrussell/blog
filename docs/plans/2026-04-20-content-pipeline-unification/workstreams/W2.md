# W2 — Blog Consumer-Enrichment Subscriber

## Objective

Build the blog-side service that subscribes to the NC envelope, maintains a
local envelope index, computes consumer fields, and emits a `stage:mined`
content-queue issue for every signal.

## Repos touched

- `~/dev/blog` — new subscriber service (likely under `tools/` as a Go
  binary; decide during kickoff)

## Deliverables

- Subscriber service reading NC Redis channels (start with a chosen subset
  from config; default `coforge:core` plus any channel explicitly enabled).
- Local envelope index keyed on `content_id` with a TTL (default 90 days).
  Implementation: SQLite unless there is a concrete reason against it.
  Stored under `~/.blog-pipeline/` or equivalent; documented in blog
  `CLAUDE.md`.
- Entity → queue-issue index (scans open `content-queue` issues).
- Entity → published-post index (scans `content/posts/**/*.md`).
- NC envelope → content-queue issue translator producing all consumer
  fields listed in `02-contracts.md`. Inference rules are explicit functions
  of envelope fields — deterministic, documented in code and in a short
  blog-side spec.
- Issue writer creating GitHub issues on `jonesrussell/jonesrussell` with
  `content-queue`, `stage:mined`, `source:<origin>`, and appropriate
  `type:*` labels.
- Blog `CLAUDE.md` gains a Gotcha section describing the subscriber, the
  index location, and how to rehydrate from NC Elasticsearch.

## Acceptance criteria

- An NC envelope arriving on a subscribed channel creates exactly one
  `stage:mined` issue with all consumer fields populated.
- Re-delivery of the same `content_id` is idempotent — no duplicate issue.
- `related_queue_issues[]` and `related_published_posts[]` reflect entities
  from the envelope, verified against at least two hand-picked examples.
- Killing the subscriber mid-stream and restarting it does not lose or
  duplicate issues.
- Local index is rehydratable from NC Elasticsearch. Document the recipe.

## Dependencies

- W1a (envelope with `entities[]`, `canonical_excerpt`, `language`).
- W1b optional — subscriber ignores `dedup_cluster_id` /
  `related_content_ids[]` if missing and uses them when present.

## Complexity

L — the biggest blog-side piece. Subscriber + two fan-in indexes +
consumer-field inference is real work.
