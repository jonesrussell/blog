# W4 — Substack as Output Channel

## Objective

Wire Substack as a permanent output channel alongside the existing Buffer
targets (X, LinkedIn, Facebook). Decide between Buffer-integrated posting,
direct-API posting, and manual-paste-with-generated-draft; implement the
chosen path.

Given Substack's API posture (see `04-risks.md` R3), the likely outcome is
**manual-paste with an automatically-generated draft**. The decision-doc
step is preserved for the record, but is time-boxed at 30 minutes. If no
API surfaces in that window, commit to manual.

## Repos touched

- `~/dev/blog` — content-pipeline skill, channels config, possibly a new
  publishing adapter
- `~/.claude/skills/content-pipeline/` — SKILL.md, channels.env,
  buffer-post.sh or a new adapter script
- `~/.claude/skills/substack-writing/` — invoked to produce the draft;
  not modified beyond what distribution wiring requires

## Deliverables

- A short decision doc in this workstream's directory capturing:
  - whether Buffer supports Substack as a channel (verify against Buffer's
    current channels list),
  - whether Substack has a usable API or email-to-post path,
  - the chosen path: Buffer / direct / manual-paste-with-draft.
  Investigation time-boxed at 30 minutes. If no API surfaces, commit to
  `manual-paste-with-draft` and stop.
- Draft generation delegates to the existing `substack-writing` skill
  at `~/.claude/skills/substack-writing/`. When an issue lists `substack`
  in `suggested_channels[]`, the distribution path invokes that skill
  with the envelope + consumer fields as input and saves the draft to the
  skill's documented output location (`~/brand/substack-issue-N.md`).
- Implementation of the chosen posting path:
  - Buffer: add a Substack channel id to `channels.env` and extend the
    content-pipeline skill to post to it.
  - Direct: write a small adapter that posts the draft.
  - Manual: after the `substack-writing` skill produces the draft, the
    distribution path pauses for operator confirmation, presents the
    draft path, and transitions the issue to `stage:distributed` on
    confirmation.
- `~/.claude/skills/content-pipeline/SKILL.md` updated with the Substack
  branch of the distribution logic (draft-generation + post-or-pause).
- `channels.env` gains a Substack entry (or a clearly documented
  manual-step marker).

## Acceptance criteria

- An issue with `substack` in `suggested_channels[]` triggers the
  `substack-writing` skill to produce a draft at the documented path.
- Distribution does not silently skip Substack. If manual paste is
  required, the pipeline says so, shows the draft path, and waits for
  operator confirmation before transitioning the issue.
- Existing Buffer targets (X, LinkedIn, Facebook) continue to work
  unchanged.
- The decision doc is committed even if the answer is "manual paste."

## Dependencies

None. Runs in parallel with everything else from day one.

## Complexity

M. The uncertainty is in Substack API access, not in wiring. The 30-minute
investigation time-box keeps the decision doc from eating the budget.
