# Blog

Hugo static site deployed to GitHub Pages at [jonesrussell.github.io/blog](https://jonesrussell.github.io/blog/). Uses the [PaperMod](https://github.com/adityatelange/hugo-PaperMod) theme.

## How This Repo Fits Into the Pipeline

This repo is the publishing surface for the content pipeline controlled by [`jonesrussell/jonesrussell`](https://github.com/jonesrussell/jonesrussell).

- **Drafts** generated by `content-produce` land here as PRs with Hugo page bundles.
- **Social bundles** reference post slugs defined here.
- **Schemas** in `schemas/` define the contracts between pipeline stages (mined-seed, curated-item).
- **Publishing is date-driven.** Hugo excludes future-dated posts. When the date arrives, the next build publishes automatically.
- **Dev.to sync** pushes published posts to Dev.to with canonical URLs pointing back here.

## Repository Structure

```
content/posts/{category}/{slug}/index.md   Page bundles (posts)
layouts/                                    Custom Hugo templates + shortcodes
static/                                     Images, favicons, OG images
assets/css/                                 Custom CSS (components, extended styles)
schemas/                                    JSON Schemas for pipeline stages
tools/devto-sync/                           Go CLI for Dev.to synchronization
scripts/                                    OG image generation, utilities
.github/workflows/
  hugo.yml                                  Build + deploy to GitHub Pages
  devto-sync.yml                            Post-deploy Dev.to push
  content-mine.yml                          Content mining automation
```

Posts are organized by category: `ai`, `architecture`, `devops`, `docker`, `general`, `go`, `laravel`, `php`, `projects`, `psr`, `waaseyaa`.

## Local Development

```bash
# Prerequisites: Hugo extended, Go, Node.js, Task (taskfile.dev)

# Dev server with drafts
task serve

# Production build
task build

# Create a new post
task new-post -- my-post-title
# Creates content/posts/{category}/{slug}/index.md

# Run checks
task check
```

The theme is a git submodule. After cloning or switching branches:

```bash
git submodule update --init
```

## Deployment

Pushes to `main` trigger `.github/workflows/hugo.yml`, which builds and deploys to GitHub Pages. No manual steps needed.

After deploy, `.github/workflows/devto-sync.yml` pushes changed posts to Dev.to.
